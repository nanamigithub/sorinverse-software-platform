# Sorinverse Platform Roadmap

## v0.1 — Structural Foundation

Goal:
Establish a stable platform skeleton and initial commercial viability.

Scope:
- Architecture layer directory complete
- Pipeline layer directory complete
- Systems registry operational
- 10–20 Apps defined
- At least 1 commercial Pack

Key Focus:
- Registry-driven UI
- App / Module / Pack composition
- No full automation yet

---

## v0.2 — Automation Activation

Goal:
Transition from UI-driven to intent-driven execution.

Scope:
- Orchestrator planning active
- Pipeline execution enabled
- Voice command input introduced
- Safety gating implemented

Key Focus:
- Reduced UI dependency
- Plan preview and approval
- Execution reporting

---

## v1.0 — Autonomous Operation

Goal:
Full realization of Sorinverse vision.

Scope:
- Voice-first operation
- Multi-step automation
- Cross-system coordination
- Self-optimizing pipelines

Key Focus:
- Outcome delivery
- Minimal interaction
- High trust autonomy

---

## Long-Term Vision

Sorinverse evolves toward:
- Invisible software
- Autonomous coordination
- Human intent amplification

Software becomes an assistant, not a tool.
